Ruby, cloud vs dedicated и проблемы rvm

Совсем недавно прочел статью с хабра, где говорят что опции компиляции ruby в rvm выбраны не самым лучшим образом. Я решил все это дело проверить на своем dev-компьютере и на облаке, которое мне дали Infobox для тестирования.

На облако мне выделили:

  • Debian 6.0
  • Intel Xeon E5-2630 @ 2.30GHz с 2 ядрами
  • 4Gb RAM

О диске инфу получить не удалось, мне либо не хватает опыта и знаний в linux, либо это особенность всех облачных решений. Сам имею:

  • OS X 10.8.2
  • Intel Core i3 @ 2.8 Ghz с 4 ядрами
  • 4x4Gb Kingston @ 1333 RAM
  • OCZ Vertex3 @ 120Gb

Тест из статьи

Облако:

# time ruby -e "count = 0; while(count < 100000000); count = count + 1; end; puts count"
100000000

real  0m4.715s
user  0m4.704s
sys   0m0.012s

Моя машина (без патча falcon, опять проблем схватил при попытке установки с этим патчем):

$ time ruby -e "count = 0; while(count < 100000000); count = count + 1; end; puts count"
100000000

real  0m2.254s
user  0m2.250s
sys   0m0.003s

Выводы простые — если нужны все возможности облака, быстрое и легкое масштабирование и прочие прелести — используем облако, если нужна хорошая производительность — берем сервер в Hetzner с SSD диском (недавно появились) и наслаждаемся скоростью.

Кстати, интересно чем обычный HDD отличается от Enterprise?

Облако будет в моем распоряжении до 1 января, до тех пор можете заказывать у меня тесты, будем проводить :)
